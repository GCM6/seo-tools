# Veris 对外营销面 SEO 方案(英文国际市场)

- 日期:2026-07-09
- 场景判定:scenario = new_build,site_stage = none(营销面尚不存在,从规划期起步)
- 目标:为 Veris 产品建立**可索引、参与谷歌排名**的英文营销面(获客 + G3 招募入口),同时工具本体维持 noindex 保护(commit `bb3d79b` 的姿态不动)
- 方法论来源:google-seo-expert skill 主线 S1→S6(逐节标注 reference 与 docs 章节);证据等级按【官方/共识/经验】标注

## 0. 架构前提(先于一切 SEO 动作)

**营销面与工具本体必须在部署层分离**,否则两边的索引/访问策略互相打架:

- 工具本体要 noindex + 访问控制(Vercel Deployment Protection),而 Deployment Protection 挡的是整个部署——营销页放同一部署会一起被挡。
- 方案:**营销站独立部署**(可同仓库不同 app,或独立轻量 Next.js 静态站),绑定主域名(如 `veris.xxx`);工具本体迁至子域(如 `app.veris.xxx`)并开 Deployment Protection。`/share` 分享页跟工具走,靠自身 noindex(已实现)。
- 域名尚未购买/确定(observed:仓库无写死生产域名,G1a runbook 用占位符)——**这是本方案第一个待决事项**。

## S1 · 可行性(← references/01,docs/seo06 §2/§6/§7)

开工前三问的回答:

1. **目标客户是否在谷歌搜?** 成立但需验证:受众是英文市场的 SEO 从业者/代理/独立站主,天然重度谷歌用户;"GEO / AI visibility / AI search optimization" 是新兴查询空间(assumed,需第一步关键词调研用真实数据确认,不得跳过)。
2. **市场体量?** 品类年轻,头部词量可能不大——对策:不押注单一新词,同时布局成熟邻接需求("AI Overviews 优化""ChatGPT 品牌提及查询"等)。若调研证实总量过低,收缩为"内容+社区获客为主、SEO 为辅",不硬投(红线:上游需求不成立不承诺 SEO 效果)。
3. **与平台标品重合?** 不重合(SaaS 工具,N/A)。

**先天缺陷预防**(三类雷,建站前规避):有关键词地图再建页;内容讲"用户怎么诊断/解决",不讲"我们功能多强";分层架构(方案页+博客)一步到位。

**独有资产(差异化根基)**:Veris 自身的探针数据与证据分级方法论 = 别人没有的第一手数据源。营销内容可发布基于真实探测的聚合研究(匿名化),这正是 S4 要求的"信息增量"与 E-E-A-T 第一手经验——**用 Veris 诊断 Veris 营销站本身**,过程即案例内容(dogfooding)。

## S2 · 选词与架构(← references/02,docs/seo5/seo1/seo06 §17/§28/§30/§19)

**流程**(先词后页,一词一页一意图):

1. 关键词地图先行:主词出发拓场景词/问题词/对比词;从 Reddit(r/SEO、r/bigseo)、X、Indie Hackers 挖"长尾中的长尾";**每个词用工具反查真实搜索量,零需求词不做**。可用 Veris 自己 + DataForSEO/Keywords Everywhere 交叉验证(S9 工具化)。
2. 词分四簇(初判,以调研结果为准修订):
   - **产品簇(→方案页/聚合页)**:AI visibility tool、generative engine optimization software、AI search monitoring 类交易意图词。
   - **诊断问题簇(→博客)**:"why doesn't ChatGPT recommend my brand"、"how to check AI Overview citations"、"llms.txt" 类问题/信息词——低 KD 先行,新站破局主力。
   - **方法论簇(→博客+方法论页)**:evidence-based SEO audit、AI share of voice measurement 类。
   - **对比/替代簇(→后期落地页)**:竞品 alternative/vs 词,权重养起后再做(新站不磕高 KD)。
3. **双擎战术**:核心词同时做"方案页(交易)+ 权威指南(信息)",TDK 与角度完全区分,防蚕食。
4. 架构:首页 + 5–8 个方案/功能页(L1)+ 方法论页(证据分级 L0–L4 本身就是差异化内容)+ 博客集群(内链全部指回方案页,无孤岛)+ 案例页(G3 陪跑产出回填)。
   - 注:docs 标配"20-30 聚合页+20-30 博客"面向外贸产品站;SaaS 营销站方案页天然少,按 5–8 起步、博客保持 15–20 篇首批,是**有意的规模适配**,分层与内链原则不变。

## S3 · 技术地基(← references/03,docs/seo06 §30/§28;勘误 C-1/C-6)

营销站(与工具本体相反,全套排名向配置都要):

- 每页独立 TDK(自然含目标词);`app/sitemap.ts` 生成并提交 GSC;robots 允许全站。
- 结构化数据:Organization / Article / SoftwareApplication / Breadcrumb。**不以富摘要为由部署 FAQ Schema**(【官方】C-1:FAQ 富摘要 2026-05-07 起全面退场;FAQPage 仍可作为给 AI 引擎的机器可读上下文,部署则问答必须与前端完全一致)。
- 性能以 **CWV 字段数据**为准:LCP ≤2.5s / INP ≤200ms / CLS ≤0.1(P75,分移动/桌面)(【官方】C-6);Lighthouse 分数只作修复线索。静态营销站用 Next SSG,达标不难。
- 单语种英文起步,**不做 hreflang/多语种**(将来加中文版再声明,严禁翻译插件泛滥)。
- OG image / favicon 全套(社媒分享面)。
- 页面层级 ≤3 层,面包屑,移动端优先。

## S4 · 内容生产(← references/04,docs/seo2/seo3/seo06 视频五六/§38/§39;勘误 C-3/C-4)

- **核心标准 = 信息增量**。Veris 的护城河内容:基于自有探针的原创数据研究(如"我们对 N 个站点 × 5 个 AI 引擎跑了同协议探测,品牌被引用率的分布是……",匿名聚合)——这是纯写手竞品(Semrush/Ahrefs 博客矩阵)给不出的第一手数据,也是自然外链磁石。
- 写法用 **AI 时代 B2B 新写法**:决策辅助视角(怎么判断自己有没有 AI 可见性问题、怎么选监测工具、指标怎么读),不堆定义(AI Overview 已直接回答定义类问题)。
- 每篇:核心词+语义相关词自然融入(不堆砌);≥1 处第一手实操见解 + ≥1 处可溯源数据;文末 CTA 指向方案页/G3 招募。
- AI 辅助初稿允许,**人工终审强制**;不设"反 AI 率"门槛(【官方】C-4:谷歌只评质量,不以 AI 生成为惩罚依据,第三方检测器无官方地位)。
- 内容重要性远高于外链是 docs 作者经验判断(【经验】C-3,禁用具体百分比),与官方 helpful content 方向一致(【共识】)。

## S5 · 冷启动与外链(← references/05,docs/seo.md/seo06 §24/§15;勘误 C-2)

- **第一指标是展示量**:上线一周展示破百 = 内容合格;破不了百先复盘内容,不上外链。
- 博客先行破局(信息词比交易页易出排名);首批 15–20 篇 + 方案页架构,不一次灌太多页。
- 外链按合规三档(【官方】C-2 强制披露):
  - **只做【官方允许】档**:社交/社区分享(LinkedIn、X、Reddit、HN——工具类产品天然适合 Show HN / r/SEO 讨论)、原创数据研究换自然引用(数字 PR)、真实供稿凭内容质量获链或加 `rel="sponsored"`。
  - **【灰色·高风险】付费 Guest Post 不进本方案**:以传权重为目的且不加 sponsored 属谷歌官方定义的 link spam,SpamBrain 持续打击;若将来评估要做,属知情风险决策,需单独批准。
  - **【违规】Web 2.0/目录批量垃圾链、工具跑链:永不做**。
- 锚文本几乎不用精准关键词;节奏自然,可接受小波动,规避触底惩罚。

## S6 · 监测与预期(← references/06,docs/seo06 §4/§10/§28)

- 上线第一天绑 GSC,判断一律以 GSC 为准;第三方工具滞后 1–2 月仅作参考。
- 见效预期(白帽前提,不承诺速成):
  - 第 1–2 月:收录 + 长尾问题词破局,展示上升;
  - 第 3–4 月:中难度词进前 10 页,优质数据内容此阶段易被 AI Overview 引用(对 Veris 是双重验证——自己的 GEO 工具测自己的 GEO 表现);
  - 第 6–9 月:核心词(如 AI visibility tool 类)争首页,取决于品类竞争度与投入。
  - 满月若近乎零展示零点击 → 复盘选词/内容/架构,不是加外链(红线)。
- 增长求健康曲线,同步做品牌声量(社区/行业曝光提升品牌搜索量 = 流量天花板)。

## S7/S8(后续阶段,本期不做)

站点满 3–6 个月后进入维护:蚕食巡检、抓取预算、内容时效更新(S7);降权信号监控(S8)。届时按 trigger-matrix 路由。

## 执行序(decision contract)

1. **【你决策】定域名与部署拓扑**(主域=营销站,子域=工具+Deployment Protection)——这同时解掉上一轮遗留的"/share vs 部署保护"取舍(share 跟工具子域走,靠 noindex;若 G3 要公开分享,营销站可放公开案例页替代)。
2. 关键词调研冲刺(1 周内):产出关键词地图(词→搜索量→KD→页面类型→层级),验证 S1 的体量假设。**此步结果决定后续投入规模**。
3. 营销站骨架:首页 + 5–8 方案页 + 方法论页 + 博客框架 + S3 技术清单全项。
4. 首批内容:15–20 篇博客(问题簇优先)+ 1 篇原创数据研究(外链磁石)。
5. 上线绑 GSC,冷启动只走合规档外链,按 S6 里程碑复盘。

## 风险与不做清单

- 最大不确定性:英文 GEO 品类的真实搜索体量(assumed,第 2 步用数据裁决;若过低则降级 SEO 投入占比)。
- 不做:付费 Guest Post(除非单独批准)、FAQ Schema 求富摘要、多语种、任何形式的批量外链/批量 AI 内容。
- 本方案不改动工具本体任何索引配置;`bb3d79b` 的 noindex 姿态在工具侧永久保留。
