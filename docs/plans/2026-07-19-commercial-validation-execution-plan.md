# Veris 4 周商业验证·实施方案

- 日期:2026-07-19
- 决策依据:盲区扫描(本会话)、采访拍板(memory `veris-4week-validation-metric`)、Reddit 痛点调研(`docs/research/2026-07-19-reddit-seo-geo-painpoints.md`)
- 一句话目标:**4 周内用陌生站主的真实行为,判定"Veris 有商业价值"为真还是假**,以此决定继续投入还是大改。

---

## 0. 判定指标(冻结区——开跑后不许改)

| 漏斗层 | 目标 | 测什么 |
|---|---|---|
| L1 触达 | ≥30 个独立站站主 | 渠道存在性 |
| L2 交站 | ≥5 人给出域名让你免费跑诊断 | 痛点存在性 |
| L3 反馈 | ≥3 人看完报告愿意 15 分钟语音 | 报告可读性/触动 |
| L4 付费意向 | ≥1 人主动问"多少钱/能持续吗/能复测吗" | 商业价值 |

- **判真**:四层全达标 → 走第 5 节"判真路线"。
- **判假**:L1≥30 但 L2<3,或 L3 达标但无人问价 → 免费都没人要,回到产品大改。
- **不可判**:L1<30 → 卡点是渠道不是产品,验证顺延、先解决触达。
- 阈值可在开跑前微调一次,开跑后冻结;中途改阈值 = 指标作废。

关键降摩擦事实(observed,本会话核实):**AI 可见度报告不需要 GSC 授权**——探针+AIO+站点审计只要域名就能跑。交站门槛从"授权 GSC"降到"发我你的网址"。GSC 相关规则(K01/K02/K06 等)作为二档加深项,仅对愿意授权者提供。

## 1. 角色分工

- **你(人)**:定阈值、找社群、发帖、聊人、做语音访谈、判定。
- **Claude(编排+执行代理)**:F1 报告开发、每个进站域名的代跑操作、报告生成、访谈记录整理、W4 数据汇总。
- 纪律:4 周内唯一开发口子 = F1 可分享报告。其余一律不写代码(违反即触发 memory 里的 4 周纪律条款)。

## 2. W1:备弹(第 1 周)

### 2.1 噪声基线检查(半天,最优先——地基)
- 对 metadocu.com 连跑两次完整诊断(什么都不改),对比两轮 GEO 指标(brand_presence / brand_sov / cited_owned_share / AIO 两率)的 run-to-run 差值。
- **通过标准**:空跑差值明显小于我们打算在报告里宣称的"有/无"结论粒度;若空跑差值大到能翻转结论,报告措辞降级为区间/方向性表述后才许对外。
- 顺带产出:单站一次诊断的真实 API 成本(探针 token+DataForSEO),用于估算 5–10 站的预算。

### 2.2 F1:可分享诊断报告(2–3 天,唯一开发项)

需求级规格(开工时由编排会话出实现计划并派发执行代理):

报告结构(站主语言,中文):
1. **头版:「AI 推荐的是你还是竞品」**——四引擎+AIO 实测结果,竞品对照(Q02 数据)。
2. **两种病因区分**:AI 检索到你但没引用(retrieved-not-cited)vs 根本没检索到 vs 爬虫被封(G01/G02)vs 内容读不到(G03)——已有数据,纯展示。
3. **「流量去哪了」**(有 GSC 授权才出):K02 + AIO 曝光卡拼接。
4. **「本月只做三件事」**:按严重度取 top3 recommendations,人话改写(N2)。
5. **「代理对账」小节**(试水,措辞用"帮你看懂代理报告",不用"抓代理"):收录数 GSC 对账 + 全站页面改动痕迹概览(contentHash)。
6. **诚信声明**:L0–L4 分级说明、"n=5 方向性样本"、"探针基于 API 与网页端可能有差异"。
7. 尾页:复测预告("4–6 周后同协议复测对比")+联系方式。

验收标准(pass-fail):
- [ ] 一个不懂 SEO 的人 10 分钟能读完头版+三件事并复述"我该做什么";
- [ ] **无行动死指标检查**:每张指标卡要么挂 recommendation,要么标"此项健康,无需行动";
- [ ] 每条结论带 claim_type 对应的中文措辞(实测/抽样/推断/假设),无一处越级;
- [ ] 可导出为单文件(静态 HTML 或 PDF),微信可发;
- [ ] 用 metadocu 真实数据渲染全文无占位符。

### 2.3 渠道与话术(1 天,你主导、我出稿)
- 列 5 个候选社群(外贸独立站微信群/知乎/小红书/相关论坛),各摸清发帖规则。
- 话术定稿,**三禁三用**(调研 §1.4/§2.4):禁"免费审计"、禁"保证排名"、慎用"GEO"自称;用"想知道 ChatGPT/AI 推荐的是你还是竞品吗"、用"可复现证据"、用"AI 可见度"。
- 备好样例报告(metadocu 版,脱敏可选)作为发帖配图/凭证。

**W1 出口检查点**:噪声检查通过 + 样例报告成品 + 话术与社群清单就绪。三者缺一不进 W2。

## 3. W2–W3:收站代跑(第 2–3 周)

节奏目标:每周触达 ≥15(两周合计 ≥30),滚动代跑。

单站 SOP(收到域名当天启动):
1. 建项目 → 跑完整诊断(无 GSC 版;对方愿授权则加深);
2. 48 小时内把 F1 报告发给对方;
3. 发报告时只问一句:"愿意花 15 分钟语音聊聊这份报告吗?"
4. 语音访谈固定五问(录音/笔记,当天归档):
   - 哪条结论是你原来**不知道**的?
   - 哪条你**会真的去做**?
   - 报告里的事,你**自己做还是希望有人代做**?(工具 vs 服务岔路,盲区⑦)
   - 这样一份报告你**愿意付多少钱**?一次性还是按月?
   - 你**还想看到什么**这里没有的?
5. 台账记录:来源渠道/触达日期/交站日期/报告发送/访谈完成/是否问价。

纪律:执行波内不改产品、不改报告结构(除 bug);所有"要不要加功能"的冲动记入 backlog 等 W4。

## 4. W4:裁决(第 4 周)

1. 汇总漏斗台账,对照第 0 节判定;
2. 访谈记录聚类:哪些痛点被复述最多、代理对账小节反应如何、工具/服务岔路答案分布;
3. 产出《验证结论报告》:判定结果 + 证据 + 路线选择;
4. 与 Reddit 调研对照:英文社区痛点在中文站主身上哪些成立哪些不成立(修正 `docs/research` 报告的可迁移性假设)。

## 5. 判定后的路线(预写好,W4 直接选)

**判真** → 按序开工(顺序在 W4 按访谈证据重排一次):
1. F2 执行资产通道接线(后端已好,UI 断链;`generated_prompts` 现为 0);
2. F3 复测噪声门补全(brand_sov/cited_owned_share/AIO 补 Wilson 或多轮)+ before/after 可分享对比页;
3. F4 逐页 TDK/H1 审计(`site_pages` 加 h1/metaDescription 列+规则扩展);
4. F5 引用稳定性指标(n=5 数据直接聚合,低垂果实);
5. N1 站内改动痕迹(contentHash 跨 run diff,并入代理对账);
6. G1a 生产部署重新上关键路径(从人肉代跑走向自助)。

backlog(等解锁条件):F6 代理对账成品 / F7 变化告警 / F8 GSC 真实 query 生成 prompt 集 / N3 GA4 结果锚定 / N4 改写敏感性采样。

**判假** → 不急着写代码:先用访谈记录做"死因分析"(是报告不可信、痛点不痛、还是执行断层),再决定大改方向;工具→服务的转向在此分支重点评估。

**不可判** → 渠道问题:换社群/换话术/找中间人背书,顺延 2 周再判;产品侧冻结不动。

## 6. 风险与停损

| 风险 | 缓解 |
|---|---|
| 噪声检查不过(空跑差值吞掉结论粒度) | 报告措辞降级为方向性表述;GEO 数字只报区间;此项不解决不对外 |
| Reddit 痛点在中文站主身上不成立 | W2 前 3 个访谈视为校准样本,话术/报告可在 W2 中期修订一次(记录修订,不算改指标) |
| 代跑成本超预算 | W1 用 metadocu 双跑实测单站成本;超预期则 L2 目标站数不变、探针档位降配 |
| "代理对账"触怒正在合作代理的站主 | 措辞固定为"帮你看懂代理报告";访谈中该节反应负面则 W2 中期撤下 |
| 你中途手痒写功能 | 本方案第 1 节纪律条款;所有功能冲动只许进 backlog 清单 |
| metadocu 单样本不足以打磨报告 | 接受:报告 v1 就是会糙,前 3 个真实站按校准样本迭代措辞(不改结构) |
