# S2 · 选词与站点架构

> 来源：docs/seo5.md（AI 自动化选词）、docs/seo1.md、docs/seo06.md §17（WriterZen+Claude）、§28（入门选词）、§30/§31（架构）、§6、§19（双擎）、§33（多站点）。
> 核心已从「看搜索量」转向「找低竞争、高转化的精准意图词」。

## 原则
- **一词一页、一种意图一个页面**：建站前先输出完整关键词地图，明确每词对应的页面类型、意图、层级（L1/L2）。从规划端杜绝关键词蚕食。
- **聚合页是权重基石**：核心转化词、高价值交易词全部布局在聚合页/产品页；博客承接信息/科普/问题类长尾词，做补充，绝不与核心页抢词。
- **先易后难**：新站权重低，先做低 KD/低竞争/黄金得分合理的长尾词、科普词积累权重，再逐步攻高难度词。高 KD 词在低权重新站很难做，养起权重后难度大降。

## 选词做法
1. 从主产品词出发，拓展 B 端采购场景词、蓝海词、转化词（可用 AI/ChatGPT 初拓）。
2. **蓝海挖掘**：不止用关键词工具的常规长尾，从推特、Reddit、行业论坛挖「长尾中的长尾」——搜索量低但竞争极低、意图精准、转化高。
3. **数据验证**：用 Keywords Everywhere / Keywords Everywhere 等插件反查真实月搜索量与趋势，剔除无搜索需求的词。无搜索量的词一律不做。
4. **分类分层**：区分商业转化词（→落地页/聚合页）与信息科普词（→博客），划 L1/L2 层级，输出站点地图。
5. **工具化**（可选，详见 S9）：WriterZen 抓数据（主题发现 + Allintitle + 黄金得分）→ Claude 智能分类归层；或 Codex/Claude Code + MCP（Firecrawl 抓 SERP、DataforSEO 取搜索量、权重 API）多工具交叉验证，封装为 Skill 复用。

## 架构做法
- **分层体系**：聚合页（L1 解决方案）→ 二级产品分类（L2）→ 博客集群，博客内链指向聚合页形成「落地页集群」。
- **双擎战术**：同一核心词同时做「商业聚合页 + 信息指南博客」，TDK/角度完全区分（如「不锈钢加工服务」vs「不锈钢加工服务选购指南」），初期双页占位、后期按需 canonical/301 合并。详见 S7。
- **用户画像先行**：落地页/博客创作前先定目标用户画像、划蓝海人群，反推内容结构。

## 多站点（同品牌多域名）
- 风险：E-E-A-T 升级后，同品牌/同联系方式/相近业务的多域名会被判权威稀释。
- 可并存条件：服务器 IP/注册信息/GSC 账号/页面框架/内容**全维度隔离** + 关键词错位 + 业务模式拆分（如 A 站 B 端批发、B 站 C 端零售）。

## 红线
- NEVER 不做关键词调研就按产品型号堆页面。
- NEVER 让博客与聚合页/产品页抢同一关键词。
- NEVER 新站一上来就死磕高 KD 大词。

## 自检
- [ ] 有完整关键词地图（词→页面类型→意图→层级）。
- [ ] 转化词在聚合页、信息词在博客，无重叠。
- [ ] 每个目标词都验证过真实搜索量。
- [ ] 新站优先低难度词。
