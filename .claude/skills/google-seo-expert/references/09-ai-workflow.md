# S9 · AI / Vibe Coding 工作流（横切）

> 来源：docs/seo2.md、docs/seo4.md、docs/seo5.md、docs/seo06.md §3（全场景）、§5（工具对比）、§34（聚合页去重）、§35（Claude Code 心得）、§36（AI 增效）、§37（内容优化工作流）、视频一。
> 定位：AI 是**执行工具**，放大成熟的人工 SEO 方法论；前提是扎实 SEO 功底，否则只会批量制造错误。

## 一、核心原则
- AI 替代精细化、重复性、高耗时工作（相关性校验、关键词匹配、内容轮替、数据汇总），人工提供规则与素材并终审。
- **AI 本身不具备主动 SEO 能力**：不会自动做关键词调研/语义覆盖/竞品分析，必须人喂规则与数据。
- 把成熟人工流程封装为自动化 **Skill** 复用：人工先完整跑一遍，让 AI 封装步骤，后续输入新核心词即可复用。
- 成本现实：人力降，但 Token/API/质检成本升，综合成本未必降；价值是质量上限 + 解放重复劳动。

## 二、落地场景
1. **选词自动化**（seo5/§17）：Codex/Claude Code 原生浏览器 + MCP 工具（Firecrawl 抓 SERP 前 100、DataforSEO 取搜索量/CPC/趋势、权重 API）多工具交叉验证，输出站点关键词架构（L1/L2）+ 难度评估 + 内容框架。
2. **内容生产**（seo2/§3/§37）：AI 做竞品调研、词频/LSI/NLP 报告、初稿、E-A-T 补充、语义完善；单篇约 90 分钟、耗大量上下文；人工终审。
3. **竞品分析与优化分**（seo2）：基于 NLP 生成 SEO 竞争分，对标超越前排竞品；提取 PAA 补内容维度。
4. **图片 SEO**（seo4/§3）：批量 WebP/压缩/统一尺寸、生成 alt、AI 配图（DALL·E 3）后处理。
5. **技术修复**（§3）：登录服务器改 .htaccess，处理 WooCommerce 筛选链接泛滥、批量 301/canonical/robots。
6. **去重与降权修复**（§34/§37）：批量抓取文章比对相似度，识别蚕食/重复，自动改写/调内链/改 HTML。
7. **数据复盘**（§9）：万级排名数据 AI 批量分类、归因、出优先级。

## 三、多智能体架构（来源 §36）
SEO 方案智能体 → 页面结构智能体 → 关键词分析智能体 → 反 AI 检测智能体 → 合规性检查智能体；协同工作，人工供素材 + 终审。

## 四、工具对比（来源 §5）
- **Claude Web Fetch（原生）**：单链接、返回 Markdown 摘要、易被反爬挡；适合轻量单篇总结。
- **Firecrawl（专业爬虫）**：批量、突破反爬、抓完整源码/站点地图/结构化数据；适合深度调研、批量竞品分析；有 API 成本。

## 五、使用心得（来源 §35）
- 桌面版（直观、单任务）vs 命令行版（多开并行、可指定工作流目录）。
- 先用规划/Problem 模式出执行计划再跑；可交叉验证（另一实例审视计划）降低出错。
- 不盲套开源工作流（多为通用/开发场景），自研专属 SEO 流程。
- 上下文容量是瓶颈：单次任务信息量过大会降质，需控制体量。

## 六、红线（合规与边界）
- NEVER 全自动化批量生成 + 自动上线低质内容（2022 日产百篇已验证行不通、会降权）。
- ALWAYS 人工终审 + 注入第一手经验/独家数据 + 数据溯源（"反 AI 率 ≤50%"已勘误，不作合规门槛，检测器结果仅作同质化参考 → references/10-official-corrections.md C-4）。
- AI 内容必须配合 NLP/LSI/E-E-A-T 多维优化，纯一键生成无竞争力。

## 自检
- [ ] AI 仅做调研/汇总/初稿/重复劳动，人工终审。
- [ ] 输出经 NLP/语义相关词/E-E-A-T 优化，人工终审通过（不以"AI 率"为门槛，见 references/10-official-corrections.md C-4）。
- [ ] 未开启「全自动生成+自动上线」。
- [ ] 工具选型匹配场景（轻量 Web Fetch / 深度 Firecrawl）。
