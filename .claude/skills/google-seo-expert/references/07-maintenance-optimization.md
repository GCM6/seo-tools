# S7 · 维护与优化

> 来源：docs/seo06.md §15（过度优化）、§22（抓取预算）、§23（波动自查）、§32（301/canonical）、§34（自动化去重）、§9（老站掉词）、§8（核心词被文章顶替）、§18（排名停滞诊断）、§19（双擎）、§33（多站点）。
> 老站运营从「做加法」转向「做减法/精细化」。

## 一、排名长期停滞的四步诊断（来源 §18）
1. **搜索意图与关键词定位偏差**：工具初判意图 → 人工到谷歌看 SERP 前排页面类型 → 错位则改（信息词别用产品页硬碰，单写信息文 + 内链；复合意图用「落地页+补充文」双页）。
2. **内链与孤岛页面**：无内链支撑的孤岛页得不到权重。用 Rank Math 内链检测找孤岛；核心聚合页配 5-6 篇相关博客内链。
3. **外链质量与数量**：识别低质/垃圾外链并拒绝；评估外链量级是否匹配词难度。
4. **技术基础与体验**：独立干净 IP；PageSpeed（重移动端）；停留/跳出等行为信号。

## 二、关键词蚕食处理（来源 §8/§23/§32）
- 表现：同一词站内多页争排名、交替波动。
- 排查：导出 GSC「搜索词↔对应页面」做匹配（可借 AI 批量识别）。
- 处理原则：按质量/数据表现，选 **canonical**（两页都有独立价值、都保留）或 **301**（彻底合并、差页跳好页）。
- 核心词被博客顶替的最优解（§8）：先把博客核心内容提炼为模块（如 Tab）并入聚合页补齐维度 → 再把博客 301 到聚合页（保留长尾价值 + 集中权重）。

## 三、301 vs canonical（来源 §32）
| | 301 重定向 | canonical |
| :--- | :--- | :--- |
| 用户访问 | 跳转到新页，看不到原页 | 原页正常访问，无感知 |
| 排名信号 | 几乎完全传给目标页 | 集中到指定主页面 |
| 页面留存 | 原页废弃 | 原页保留 |
- 301 适用：内容高度重复彻底合并 / 旧页删除改版 / 确认无独立价值。
- canonical 适用：两页有重叠但都有价值、不想删（如双擎信息页指向落地页）。
- **跨域名 canonical 几乎无效**；以上均为站内操作。

## 四、抓取预算瘦身（来源 §22/§26）
- 谷歌对每站抓取有上限，与权重正相关；无效页泛滥会致新页不收录、老页掉排名。
- 浪费源：无效小语种页、Tag/分类页泛滥、404/失效链接、低质无排名页、过长 301 链。
- 瘦身：① Tag/冗余分类页设 noindex；② GSC 导近 3 月数据筛无展示无点击页 → 可合并的 canonical、无价值的删除、有潜力的重写；③ 修 404（有价值旧链 301）、清理过长重定向链。

## 五、过度优化收口（来源 §15）—— 越优化越差的根因
- 基础过度优化：关键词堆砌、单一精准锚文本、短时间批量外链、全站套同一优化模板。
- 深层过度优化（更隐蔽）：
  1. **泛流量过多**：控制泛流量占比 20-30%，70-80% 围绕精准转化词。
  2. **页面新增过度**（内容垃圾场）：超出权重承载会稀释抓取、被判内容工厂 → 降更新频率，从「增量」转「提质」（如设备站改 3-5 天 1 篇，优化老页）。
  3. **外链增长刻意**：放缓节奏，靠优质内容自然获引用外链。

## 六、老站存量维护（来源 §9）
- 多源导出（Semrush 全量排名变化 + GSC 全量搜索）交叉验证 → AI 批量识别丢失词、归因（覆盖不足/蚕食/质量下降）、输出优先级 → 针对性：内容补全 / 301 合并 / 内容更新重写。
- 时效性内容（榜单/评测/数据）过期会自然掉排名 → 更新数据 + 改发布日期重新提交。

## 红线
- NEVER 排名停滞就盲目加内容/加外链——先按四步诊断定位卡点。
- NEVER 放任关键词蚕食。
- NEVER 老站继续无脑堆量。

## 自检
- [ ] 停滞问题已四步诊断定位。
- [ ] 蚕食用 canonical/301 正确处理（站内）。
- [ ] 抓取预算已瘦身（noindex/删低质/修 404）。
- [ ] 泛流量占比 ≤30%，更新转向提质。
