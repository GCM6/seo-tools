## 1. 核心原则

1. **官方优先**：docs 反映其撰写时点的实践经验；与谷歌官方政策/文档冲突时，一律以官方为准，并向用户提示时点差异（对齐 `.meta/GOVERNANCE_PROFILE.md` §5）。
2. **证据分级**：本 skill 的所有主张按三级标注——【官方】（谷歌/微软等官方文档或公开声明）、【共识】（多来源独立研究一致）、【经验】（docs 作者单一来源实践估计）。【经验】级主张不得以事实口径对外输出。
3. **勘误单一真源**：勘误条目只在本文完整定义；受影响 reference 用 `〔官方勘误 → references/10-official-corrections.md C-N〕` 行内标注。
4. **合规风险披露**：凡官方明确列为违规/高风险的操作（如买卖传权链接），给方案时必须披露官方定性与后果，由用户知情决策。

## 2. 勘误条目

### C-1 FAQ Schema 富摘要已全面退场【官方】

- **docs 原主张**（seo06 §20，时点约 2025）："FAQ Schema 性价比高且当前稳定生效"。
- **官方现实**（developers.google.com/search/updates，2026-07-03 核实）：FAQ 富摘要 2023-08 起仅限权威政府/健康站点（"only shown for well-known, authoritative government and health websites"）；**2026-05-07 起对所有站点停止展示**（"This feature will no longer appear in Google Search starting May 7, 2026"）；2026-06 官方文档删除。HowTo 富摘要 2023-09 已移除。
- **修正口径**：❌ 不得再以"能拿富摘要/提升点击率"为由建议部署 FAQ Schema。✅ FAQPage 仍是有效 schema.org 类型，已部署的保留无害；新价值定位是**为 AI 引擎提供机器可读上下文**——微软官方确认 schema 喂给 Bing LLM/Copilot 用于内容理解（Fabrice Canel，SMX Munich 2025-03）【官方·机制层】，但"提升 AI 引用率 X 倍"类数字均无对照实验【经验以下，不得引用】。"Schema 与前端内容完全一致"红线不变且更重要。

### C-2 付费外链与主动制造低质外链属官方违规【官方】

- **docs 原主张**（seo.md，时点约 2025）：外链标准三步"社交分享 → Web 2.0/目录基础外链 → 付费 Guest Post（120-150 条、单条约 500 元案例）"。
- **官方现实**（Google spam policies - link spam）：**为传递排名信号而买卖链接属 link spam**；付费/广告链接必须 `rel="sponsored"`（标注后不传权重）；主动批量制造低质链接同属 link scheme，SpamBrain 持续打击。
- **修正口径**：外链手段按三档披露，给方案时必须写明档位——
  - 【官方允许】自然获得的引用外链、数字 PR、真实供稿（链接加 `sponsored/nofollow` 或凭内容质量自然获链）、社交分享。
  - 【灰色·高风险】不加 `sponsored` 的付费 Guest Post、以传权重为目的的批量客座投稿——官方定性违规，实践普遍，属"知情风险操作"，须向用户披露官方定性与 SpamBrain 处罚可能。
  - 【违规·不建议】Web 2.0/目录类批量"垃圾外链"、链接农场、工具自动跑链——docs 自述其目的是"稀释锚文本降低作弊嫌疑"，即规避检测，风险最高。
- docs 的量级参考（年 80-100+ 条等）保留为【经验】级行业观察，不作为建议输出。

### C-3 "内容占排名权重 70-80%、外链 20-30%"为经验估计【经验】

- **docs 原主张**（seo06 视频五等）：以确定数字表述内容/外链权重占比。
- **官方现实**：谷歌从未公布任何排名因子权重比例。
- **修正口径**：只能表述为"docs 作者的实践经验认为内容重要性远高于外链"；对外输出禁止引用具体百分比。策略结论（内容优先于外链投入）本身与官方"helpful content 为核心"方向一致【共识】，可保留。

### C-4 "反 AI 率"指标缺乏官方依据【经验】

- **docs 原主张**（seo06 视频五/§39）：AI 内容需将第三方检测器的"AI 率"降到 50% 以下。
- **官方现实**（Google Search guidance on AI-generated content）：谷歌明确**不以"是否 AI 生成"为惩罚依据，只评估内容质量与有用性**（"rewarding high-quality content, however it is produced"）；谷歌从未采用或认可第三方 AI 检测器，此类检测器误报率高、无权威校准。
- **修正口径**：❌ 不得把"反 AI 率 ≤50%"当作合规门槛输出。✅ 保留其内核：人工终审 + 注入第一手经验/独家数据/来源溯源（这是官方 E-E-A-T 方向【官方】）；第三方 AI 检测结果至多作为"内容同质化程度"的参考信号【经验】。

### C-5 "LSI 关键词"是伪技术术语【官方】

- **docs 原主张**：以"LSI 关键词"命名相关词拓展方法。
- **官方现实**：谷歌官方（John Mueller）明确表态 "There's no such thing as LSI keywords"——谷歌不使用 LSI（潜在语义索引，1980 年代文献检索技术）。
- **修正口径**：概念内核（覆盖语义相关词、避免堆砌核心词）正确且保留【共识】；对外输出改称"**语义相关词**"，如沿用"LSI"须注明是行业俗称、非谷歌机制。术语条目见 `references/00-glossary.md`。

### C-6 页面速度标准应采用 Core Web Vitals 口径【官方】

- **docs 原主张**（seo06 §30）："移动端满载 ≤2.5s、桌面 ≤1s 为优"。
- **官方现实**：现行官方性能框架是 **Core Web Vitals 字段数据**——LCP ≤2.5s / INP ≤200ms / CLS ≤0.1（P75 分位，CrUX 真实用户数据；INP 已于 2024-03 取代 FID）。排名使用的是 CWV 字段数据（轻量级信号）；Lighthouse/PSI 的 0-100 分是实验室诊断值，**不是排名输入**。"满载时长"是过时口径。
- **修正口径**：性能建议统一以 PSI 的 CWV 字段数据（分移动/桌面）为判定标准，Lighthouse 分数仅作修复线索；docs 的优化手段（WebP、压缩、CDN、代码精简）仍有效【共识】。
