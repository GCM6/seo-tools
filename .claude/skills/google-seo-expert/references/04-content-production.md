# S4 · 内容生产体系

> 来源：docs/seo2.md、docs/seo3.md、docs/seo06.md 视频五（AI 内容）/视频六（博客工作流）/§14（博客经验）/§19（双擎）/§38（NLP）/§39（E-E-A-T）/§21（获客）/§25（排名因素）。
> 内容是绝对核心，docs 作者经验认为其重要性远高于外链（原文"70-80% vs 20-30%"为经验估计，非官方数字，对外输出禁用具体百分比〔官方勘误 → references/10-official-corrections.md C-3〕）。落地顺序：先内容定稿，再页面设计/前端，最后嵌入聚合页体系。

## 核心认知
- **从「我想告诉用户什么」转向「用户会搜什么、想问什么」**。公司动态/展会/研发理念这类无搜索量内容一律不做。
- 博客占 60-70% 自然流量，是前期破局主力（利他信息型，比交易型落地页更易快速出排名）。
- 内容能否做 SEO 的核心标准是**信息增量**——相比已有内容是否提供更多有价值的信息/数据/观点。

## 博客写作 SOP（当前版）
1. **前置研究**：判断搜索意图分类、分析 SERP 前排页面类型、用 PAA/Also Asked 拓展用户真实问题、验证内容方向。
2. **关键词体系**：核心词 + 语义相关词（行业俗称 LSI，非谷歌机制，术语勘误见 references/00-glossary.md）+ NLP 语义词三类，自然融入（符合 TF-IDF/语义规则），不堆砌。
3. **内容框架**：按文章类型（问题型/信息/对比/案例/top 榜单/权威指南）匹配结构；底部植入内链指向核心聚合页，避免孤岛。
4. **主题群**：按主题集群规划内链，把相关文章串联，共同支撑核心页权重。
5. **证据与质检**：所有数据/观点标注来源、有证据等级；多轮质检（质量评分、可读性、边界合规、来源复核）。
6. **AI 落地**：把成熟人工流程封装为 AI 工作流（Skill），AI 做调研/汇总/初稿，人工做终审与质量把控（详见 S9）。

## B2B 文章 AI 时代写法（来源 docs/seo3.md）
- 旧的「是什么-类型-组成」堆定义写法已失效（AI Overview 直接输出定义）。
- 新写法 = **采购决策辅助**，站在买家视角：先做目标国用户画像；在定义之外补三类决策内容——
  - 采购方真实疑虑与常见顾虑解答；
  - 产品真实应用场景还原；
  - 参数/指标对采购的实际价值（「这个参数对买家意味着什么」）。
- 分层布局：成交词在聚合页，决策类深度内容（供应商筛选、选型步骤、行业指南）拆为博客归资源页。

## NLP 内容优化（来源 §38）
- 三代进化：关键词堆砌（已失效）→ LSI 自然分布 → **NLP 语义深化**（加行业术语、参数原理、实体变体、场景化描述，从「介绍产品」升级为「讲解行业知识」）。
- 落地：先列 10-20 个行业专属术语/实体词，结合 LSI 清单规划大纲，自然融入；配下沉博客与内链做语义延伸。
- 红利：NLP 优化的专业内容更易被 AI Overview 引用。

## E-E-A-T 落地（来源 §39）
- 强化「经验性」：加第一人称实操经验、选型心得、踩坑总结、场景化建议，把「说明书」变「从业者分享」。
- 博客：每篇 ≥1 处个人实操见解 + ≥1 处权威文献/数据来源。
- 聚合页：真实应用场景 + 客户痛点与解决方案 + 选型/避坑建议 + 资质证书/真实案例。

## AI 内容合规（来源 视频五 / §27误区4）
- 谷歌不反对 AI 辅助，反对的是**低质量、无价值、无人工审核的批量生成**。
- 纯 AI 直出无法获得好排名：必须人工审核 + 行业经验/案例/独家数据注入 + 数据溯源。
- **"反 AI 率"指标勘误**〔→ references/10-official-corrections.md C-4〕：谷歌官方明确只评估内容质量、不以"是否 AI 生成"为惩罚依据，且从未认可第三方 AI 检测器；docs 的"AI 率降到 50% 以下"不得作为合规门槛输出。真正的门槛是：人工终审 + 注入第一手经验/独家数据 + 来源溯源；检测器结果至多作同质化参考。
- 初始优质内容 SEO 优化分可达 88-90，再经多轮迭代 + 人工终审。

## 落地页与转化（来源 视频三 / §21）
- 落地页必须讲清**应用场景 + 能解决的具体问题**，而非只罗列参数；用户最关心的放首屏。
- 针对采购决策痛点设计：成本、选型标准、售后保障、定制能力。
- 博客文末植入产品 CTA，看完即可跳转咨询；博客内链向聚合页传权重。
- 获客导向：从业务需求/用户画像倒推关键词→内容→页面，形成「博客种草 → 聚合页转化」闭环。流量再大无询盘即无效。

## 红线
- NEVER 一键批量生成、无人工终审的 AI 内容。
- NEVER 写无搜索量的自嗨内容（公司动态/展会）。
- NEVER 堆定义、无数据、无决策导向的旧式 B2B 文章。
- ALWAYS 人工终审 + E-E-A-T + 数据溯源。

## 自检
- [ ] 每篇内容围绕用户搜索需求，有信息增量。
- [ ] 含 LSI + NLP 语义覆盖，无堆砌。
- [ ] 有 E-E-A-T 要素（经验 + 来源）。
- [ ] AI 内容经人工终审，注入第一手经验与来源溯源（不以"AI 率"为门槛，见 references/10-official-corrections.md C-4）。
- [ ] 内链指向聚合页，无孤岛；落地页讲场景/价值 + CTA。
